## 3. Harmonic Notch Filter Parameters (노치 필터 파라미터 확정)

Motor fundamental ~49 Hz, blade pass ~98 Hz가 최대 진폭으로 확인됨.
HMNCS=3으로 두 피크를 동시 억제, MODE=3(ESC CAN telemetry)으로
RPM 기반 실시간 추적 적용.

| Parameter | Value | 비고 |
|-----------|-------|------|
| INS_HNTCH_ENABLE | 1 | — |
| INS_HNTCH_MODE | 3 | ESC CAN telemetry 실시간 RPM 추적 |
| INS_HNTCH_FREQ | 40 | 하한 가드 — hover 49Hz의 ~80% |
| INS_HNTCH_HMNCS | 3 | 1st(49Hz) + 2nd(98Hz) 동시 노치 |
| INS_HNTCH_BW | 20 | 노치 대역폭 |
| INS_HNTCH_ATT | 40 dB | 감쇀량 |

### FTN1 Tracking Result (추적 결과 — After 로그)
- PkAvg mean = **99.7 Hz** -> ESC RPM 기반 정상 추적 확인
- FtX mean = **93.1 Hz**
- 하한 가드(40 Hz) 이하 이탈 없음

## 4. Loiter Parameters (로이터 파라미터 확정)

건물 외벽 근접 + 200bar 수압 반력 환경 특성을 반영.
기울기 억제를 최우선으로 설정하되, 호스 장력으로 인한
스틱 중립 후 밀림 현상은 BRK_ACCEL로 억제 확인.

| Parameter | Value | 비고 |
|-----------|-------|------|
| LOIT_SPEED | 250 cm/s | 2.5 m/s — 작업 속도 기준 |
| LOIT_ACC_MAX | 250 cm/s2 | 기울기 억제 우선 |
| LOIT_ANG_MAX | 15 deg | 건물 근접 안전 제한 (핵심) |
| LOIT_BRK_ACCEL | 150 cm/s2 | 호스 장력 밀림 억제 확인 |
| LOIT_BRK_DELAY | 0.3 s | — |
| LOIT_BRK_JERK | 500 cm/s3 | 펌웨어 최솟값(500) 준수 — 300에서 수정 |

### Tuning Rationale (결정 근거)
- LOIT_ANG_MAX=15: 수압 반력 + 건물 근접 환경에서 과도한 lean 방지
- LOIT_BRK_ACCEL=150: 스틱 중립 후 서서히 밀리는 현상 억제 확인
- LOIT_BRK_JERK=500: 펌웨어 하한 위반(300) 수정 -> 제동 해제 반응성 개선

## 5. WP Speed Tuning (속도 튜닝)

3포지션 스위치로 ZigZag 세척 패턴 속도를 즉시 전환하는 운용 방식.

| Parameter | Value | 비고 |
|-----------|-------|------|
| TUNE | 10 | WP Speed |
| TUNE_MIN | 50 cm/s | 0.5 m/s — 정밀 작업 |
| TUNE_MAX | 200 cm/s | 2.0 m/s — 일반 작업 |

| Switch Position | PWM | WP Speed |
|-----------------|-----|----------|
| Low | ~1000us | 0.5 m/s |
| Mid | ~1500us | 1.25 m/s |
| High | ~2000us | 2.0 m/s |

## 6. Operational Notes (운용 환경 특이사항)

- **Hose tension (호스 장력):** 300m 수압 호스(200bar) 반력이 지속적으로 기체를 당김
  -> 위치 유지 복원력 추가 튜닝 필요 시 PSC_VELXY_P 검토
- **Building proximity (건물 근접):** LOIT_ANG_MAX 엄격 제한 필수
- **ESC connection:** PWM ESC + DroneCAN 연결
  -> MODE=3 RPM telemetry 정상 수신 확인
